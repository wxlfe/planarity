# AGENTS.md

## Project Shape
- Single Flutter app; primary app code, game logic, UI, Firebase integration, and test-exposed helpers are currently concentrated in `lib/main.dart`.
- Entrypoint is `lib/main.dart`; VS Code launch config also points there.
- Tests live in `test/widget_test.dart` and import `package:planarity/main.dart` directly.

## Commands
- Install dependencies: `flutter pub get`.
- Format edited Dart files: `dart format <paths>`.
- Analyze: `flutter analyze`.
- Run all tests: `flutter test`.
- Run the focused test file: `flutter test test/widget_test.dart`.
- Run one named test: `flutter test test/widget_test.dart -n '<test name>'`.
- Build web like CI: `flutter build web`.

## Verification Order
- For normal code changes, run `dart format` on touched Dart files, then `flutter analyze`, then relevant `flutter test`.
- For deploy-sensitive web changes, also run `flutter build web`; CI does only `flutter pub get` and `flutter build web`.

## Localization
- Source strings are `lib/l10n/app_*.arb`; generated files are in `lib/l10n/generated/`.
- Do not hand-edit `lib/l10n/generated/*`; after ARB changes run `flutter gen-l10n`.
- `lib/l10n/app_localizations.dart` is the local wrapper/extension layer around generated localizations.

## Firebase And Data
- `lib/firebase_options.dart` is generated by FlutterFire and tracked; platform service files such as `android/app/google-services.json` and Apple `GoogleService-Info.plist` files are ignored.
- Firestore rules are tracked in `firestore.rules`; keep user document fields in sync with app writes in `lib/main.dart`.
- Widget tests avoid live Firebase by constructing `PlanarityApp` directly and using local/shared-preferences state.

## Web And Deploy
- GitHub Actions deploy workflow builds Flutter web, then copies `vercel.json`, `web/.well-known/apple-app-site-association`, and `web/.well-known/assetlinks.json` into `build/web` before Vercel deploy.
- `vercel.json` owns clean URL rewrites for `/privacy` and `/support` plus no-store headers for app shell and well-known files.
- `web/index.html` intentionally unregisters service workers and clears caches before loading Flutter; avoid reintroducing PWA caching without checking that behavior.

## Tests
- Widget tests commonly call `SharedPreferences.setMockInitialValues({})` before pumping `PlanarityApp`.
- Existing widget tests pump `const Duration(milliseconds: 100)` after `PlanarityApp` to allow startup state to settle.
